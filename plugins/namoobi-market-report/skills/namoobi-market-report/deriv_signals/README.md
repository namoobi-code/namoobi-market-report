# 파생 포지셔닝 → 현물 선행신호 (KOSPI200 / S&P500 / Nasdaq100)

파생시장(선물·옵션) 포지셔닝을 표준화(z-score)해 **신호 발생일 이후 1/3/5일 현물수익률**로
선행성을 검증하고, 결과를 SQLite DB로 축적·매일 자동 갱신하는 예시 파이프라인.

- 데이터: **전부 무료 소스**(yfinance, CFTC COT 직접 다운로드) → **API 키 불필요**
- 산출: `deriv_signals.db` (SQLite) + `outputs/*.csv` + 콘솔 리포트

---

## 1. 지표 구성 (요청 항목 매핑)

| 요청 항목 | 컬럼 | 산출 방식 |
|---|---|---|
| 현물 수익률 | `spot_ret` | 현물지수 종가 일간수익률 |
| 선물 수익률 | `fut_ret` | 선물 연속물 일간수익률 |
| 베이시스 | `basis_bp` | (선물/현물 − 1) × 10,000 (bp) |
| 선물 순포지션 | `lev_net`, `asset_mgr_net` | COT: (롱−숏), 레버리지펀드·자산운용사 |
| OI 변화 | `oi_chg_w` | COT 주간 미결제약정 증감 |
| 풋콜비율 | `pcr_oi`, `pcr_vol` | 옵션체인 풋/콜 미결제·거래량 비 |
| 옵션 델타/IV 스큐 | `iv_skew_25d`, `delta_imbalance` | 25Δ 풋IV−콜IV, 델타가중 풋/콜 |

→ 각 지표를 **롤링 z-score**로 표준화 → `|z| ≥ 1.5` 신호 이벤트 → **1/3/5일 후 현물수익률** 부착.

지표 의미·해석은 **`해석가이드.md`** 참조.

## 2. 1년치 데이터 확보 가능성 (실측 결과)

| 지수 | 현물 | 선물·베이시스 | 순포지션·수급 | PCR | 델타·IV스큐 |
|---|---|---|---|---|---|
| **S&P500 (SPX)** | O ^GSPC | O ES=F | O COT(주간) | △ 스냅샷 | △ 스냅샷 |
| **Nasdaq100 (NDX)** | O ^NDX | O NQ=F | O COT(주간) | △ 스냅샷 | △ 스냅샷 |
| **KOSPI200** | O KODEX200 ETF | △ 스냅샷 | **O 외국인·기관 일별(네이버)** | ✕→data.go.kr | ✕→data.go.kr |

- **O(1년 백필 가능)**: 미국 현물·선물·베이시스·COT, **KOSPI200 현물·외국인·기관 순매수** → 즉시 1년치 적재.
- **△(전진 축적)**: 옵션 PCR·IV스큐·델타 및 KOSPI200 선물 베이시스는 **당일 스냅샷만** → 매일 1행씩 축적.
- **✕→확장**: KOSPI200 옵션 PCR/IV는 무료 스냅샷 불가 → 공공데이터포털 API로 확장(아래 3-3).

## 3. 실행

```bash
pip install -r requirements.txt

python3 run_backfill.py     # 1년치 백필(최초 1회)
python3 daily_update.py     # 매일 증분 갱신 + 당일 리포트
```

### 3-1. KOSPI200 현물
Yahoo `^KS200`·네이버 KPI200 지수 feed는 이상치가 많아 **KODEX 200 ETF(069500.KS)**로 현물 대용(추종오차 미미).

### 3-2. KOSPI200 수급 포지셔닝 — 무료·자격증명 불필요 (핵심 방법)
`ingest_kr.py`가 네이버 금융 '투자자별 매매동향'에서 **외국인·기관 일별 순매수(억원)**를 1년치 수집한다.
한국에서 외국인 순매수는 대표적 **일별** 선행지표다(미국 COT는 주간 → 韓이 더 촘촘·즉각적).
저장 시 외국인→`lev_net`, 기관→`asset_mgr_net` 슬롯에 매핑되어 동일 z-score/검증 파이프라인을 그대로 탄다.

> 실측(예시): 외국인 순매수 z≥+1.5 이후 **5일 평균 +5.97%, 적중 87%** (표본 15, IC 0.21).
> 외국인 순매수가 KOSPI200 최강 선행신호임을 데이터가 확인.

### 3-3. KOSPI200 선물 베이시스·옵션 확장
- 선물 최근월 스냅샷: `ingest_kr.kospi200_futures_snapshot()` (네이버 당일값) → 베이시스 참고용.
- **1년 소급·옵션 PCR/IV**: 무료 확장은 **공공데이터포털(data.go.kr) '금융위원회_파생상품시세정보'**
  무료 API 키 권장(KOSPI200 선물·옵션 일별 시세/미결제). 또는 KRX 계정(`KRX_ID`/`KRX_PW`)+pykrx.

## 4. DB 스키마 (SQLite)

| 테이블 | 내용 |
|---|---|
| `instruments` | 지수 참조(티커·COT명) |
| `prices_daily` | 현물/선물/VIX 종가 |
| `positioning_weekly` | COT 주간 순포지션·OI |
| `options_daily` | 옵션 PCR·IV스큐·델타(일별 스냅샷 누적) |
| `indicators_daily` | 통합 일별 패널 + 1/3/5일 선행수익률 |
| `zscores_daily` | 지표별 롤링 z-score |
| `signal_events` | `|z|≥1.5` 신호 이벤트(롱포맷) + 선행수익률 |
| `validation_summary` | 지표×방향별 적중률·평균수익률·IC |
| `update_log` | 실행 로그 |

`outputs/signal_validation_summary.csv`, `outputs/signal_events.csv` 로도 내보내짐.

## 5. 매일 자동 업데이트
`daily_update.py`가 최근 구간을 재수집 → 옵션 스냅샷 1건 누적 → z/신호/검증 재계산 → 게시.
Cowork 예약작업으로 평일 아침 자동 실행되도록 등록되어 있다(시간 변경 가능).

## 6. 한계·주의
- COT는 **주간**(화요일 기준, 금요일 공표) → 일별 패널에선 전진보간(ffill). 신호가 며칠 지속됨.
- 옵션 지표는 과거 소급 불가 → 표본이 쌓이기 전(30거래일)엔 z/신호 미생성.
- 베이시스는 만기·배당·롤에 따른 구조적 성분 포함 → z(자기 최근평균 대비)로 완화하나 롤 점프 잔존.
- `Z_WINDOW=60`은 1년 표본 기준값. 데이터가 쌓이면 120/252로 확대 권장.
- 통계는 표본기간 의존·다중검정 위험 → 투자권유 아님, 리서치 예시.
